## PURPOSE

This document defines what is included and excluded from testing to manage 
project scope and set clear expectations for this portfolio demonstration.

---

## IN SCOPE ✅

### Core E-Commerce Flow
**Justification:** These features represent the critical purchase path that 
demonstrates essential QA skills.

#### 1. Login Module
- Valid/invalid credentials
- Empty field validation
- Locked account handling
- SQL injection awareness

#### 2. Inventory Module
- Product list display
- Product sorting (4 options)
- Product detail navigation
- Image and text verification

#### 3. Cart Module
- Add items to cart
- Remove items from cart
- Cart badge counter
- Multiple item management

#### 4. Checkout Module
- Checkout initiation
- Form field validation
- Order confirmation
- Empty field handling

---

## OUT OF SCOPE ❌

### Navigation Elements
**Reason:** Not part of core purchase flow

- Hamburger menu (About, Logout, Reset)
- Social media links (Twitter, Facebook, LinkedIn)
- Footer links

### Technical Testing
**Reason:** Requires specialized tools/environment

- API testing
- Database testing
- Backend validation
- Server-side security

### Extended Testing
**Reason:** Time constraints for portfolio project

- Mobile responsive testing
- Cross-browser testing (Firefox, Safari, Edge)
- Accessibility testing (WCAG)
- Internationalization/Localization
- Print functionality
